# Security & Privacy Compliance Standards - Europe

## Overview

This document provides a comprehensive listing of all security and privacy compliance standards and certifications relevant for Europe (EU/EEA).

---

## 1. Core Data Protection Regulations

### GDPR (General Data Protection Regulation)

**Applicability:** Any organization processing personal data of EU/EEA residents, regardless of location

**Key Principles (Article 5):**

| Principle | Description |
| --- | --- |
| Lawfulness, Fairness, Transparency | Clear legal basis; transparent processing |
| Purpose Limitation | Collect for specified, explicit, legitimate purposes |
| Data Minimization | Adequate, relevant, limited to necessary |
| Accuracy | Keep data accurate and up to date |
| Storage Limitation | Keep only as long as necessary |
| Integrity and Confidentiality | Ensure appropriate security |
| Accountability | Demonstrate compliance |

**Legal Bases for Processing (Article 6):**

| Legal Basis | Description |
| --- | --- |
| Consent | Freely given, specific, informed, unambiguous |
| Contractual Necessity | Necessary to perform a contract |
| Legal Obligation | Required by law |
| Vital Interests | Protect someone's life |
| Public Interest | Official authority or public interest task |
| Legitimate Interests | Balanced against data subject rights |

**Data Subject Rights:**

| Right | Article | Description |
| --- | --- | --- |
| Right to be Informed | 13-14 | Privacy notice at collection |
| Right of Access | 15 | Obtain copy of personal data |
| Right to Rectification | 16 | Correct inaccurate data |
| Right to Erasure ("Right to be Forgotten") | 17 | Request deletion |
| Right to Restrict Processing | 18 | Limit how data is used |
| Right to Data Portability | 20 | Receive data in machine-readable format |
| Right to Object | 21 | Object to processing |
| Rights Related to Automated Decision-Making | 22 | Human review of automated decisions |

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| Data Protection Officer (DPO) | Required for: public authorities, large-scale monitoring, special categories |
| Data Protection Impact Assessment (DPIA) | Required for high-risk processing |
| Records of Processing Activities | Document all processing activities |
| Privacy by Design and Default | Build privacy into systems |
| Breach Notification | 72 hours to supervisory authority |
| Cross-Border Transfers | Adequacy decisions, SCCs, BCRs, or other mechanisms |
| Vendor Management | Data Processing Agreements required |
| Security Measures | Appropriate technical and organizational measures |

**Special Categories of Data (Article 9):**

- Racial or ethnic origin
- Political opinions
- Religious or philosophical beliefs
- Trade union membership
- Genetic data
- Biometric data (for identification)
- Health data
- Sex life or sexual orientation

**Processing requires explicit consent or specific exceptions**

**Penalties:**

- Tier 1: Up to EUR 10 million or 2% of global annual revenue
- Tier 2: Up to EUR 20 million or 4% of global annual revenue

### ePrivacy Directive (2002/58/EC)

**Applicability:** Electronic communications sector across EU

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| Cookie Consent | Prior consent for non-essential cookies |
| Direct Marketing | Opt-in for electronic marketing |
| Traffic Data | Confidentiality of communications |
| Location Data | Consent for processing location data |
| Calling Line Identification | Controls for caller ID |
| Spam Protection | Unsolicited communications restrictions |

**Cookie Categories:**

| Category | Consent Required |
| --- | --- |
| Strictly Necessary | No |
| Performance/Analytics | Yes |
| Functionality | Yes |
| Targeting/Advertising | Yes |

## 2. Cybersecurity Regulations

### NIS2 Directive (Network and Information Security Directive 2)

**Applicability:** Essential and important entities across critical sectors

**Transposition Deadline:** October 17, 2024 (many member states still pending as of 2025)

**Sectors Covered:**

| Essential Entities | Important Entities |
| --- | --- |
| Energy | Postal and courier services |
| Transport | Waste management |
| Banking | Chemicals |
| Financial market infrastructure | Food production/processing |
| Health | Manufacturing |
| Drinking water | Digital providers |
| Wastewater | Research |
| Digital infrastructure | |
| ICT service management (B2B) | |
| Public administration | |
| Space | |

**Size Thresholds:**

| Category | Employees | Annual Turnover | Balance Sheet |
| --- | --- | --- | --- |
| Medium+ | 50+ | EUR 10M+ | EUR 10M+ |
| Large | 250+ | EUR 50M+ | EUR 43M+ |

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| Risk Management | Policies on risk analysis and information security |
| Incident Handling | Detection, response, recovery procedures |
| Business Continuity | Backup, disaster recovery, crisis management |
| Supply Chain Security | Security in supplier relationships |
| Security in Acquisition | Security in network/system acquisition, development, maintenance |
| Effectiveness Assessment | Policies to assess cybersecurity effectiveness |
| Cyber Hygiene | Basic cyber hygiene practices and training |
| Cryptography | Policies on cryptography and encryption |
| HR Security | Human resources security, access control |
| Authentication | Multi-factor or continuous authentication |

**Incident Reporting:**

| Timeline | Requirement |
| --- | --- |
| 24 hours | Early warning (significant incident) |
| 72 hours | Incident notification (with initial assessment) |
| 1 month | Final report (root cause, mitigation, cross-border impact) |

**Penalties:**

- Essential entities: Up to EUR 10 million or 2% of global annual turnover
- Important entities: Up to EUR 7 million or 1.4% of global annual turnover
- Management liability for non-compliance

---

### DORA (Digital Operational Resilience Act)

**Applicability:** Financial sector entities and their ICT third-party providers

**Effective Date:** January 17, 2025

**Key Requirements (Five Pillars):**

| Pillar | Requirements |
| --- | --- |
| ICT Risk Management | Comprehensive ICT risk framework, governance, policies, procedures |
| ICT Incident Management | Classification, detection, response, root cause analysis |
| Digital Resilience Testing | Regular testing (basic and advanced), TLPT for significant entities |
| ICT Third-Party Risk | Risk assessment, due diligence, contractual requirements, exit strategies |
| Information Sharing | Voluntary cyber threat intelligence sharing |

**ICT Incident Reporting Timelines:**

| Timeline | Requirement |
| --- | --- |
| 4 hours | Initial notification (from classification) |
| 72 hours | Intermediate report |
| 1 month | Final report |

**Penalties:** Up to EUR 1 million per day for non-compliance

---

### Cyber Resilience Act (CRA)

**Applicability:** Products with digital elements placed on EU market

**Status:** Published December 2024; obligations phased in through 2027

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| Security by Design | Secure development lifecycle |
| Vulnerability Handling | Coordinated vulnerability disclosure process |
| Security Updates | Provide updates for product lifetime (minimum 5 years) |
| Secure Configuration | Secure default configuration |
| Incident Reporting | Report actively exploited vulnerabilities within 24 hours |
| CE Marking | Demonstrate conformity for CE marking |
| SBOM | Software Bill of Materials |

**Penalties:** Up to EUR 15 million or 2.5% of worldwide annual turnover

## 3. AI Regulation

### EU AI Act

**Applicability:** AI systems placed on EU market or used within EU

**Effective Dates:**

| Date | Provisions |
| --- | --- |
| August 1, 2024 | Entry into force |
| February 2, 2025 | Prohibited AI practices effective |
| August 2, 2025 | General-purpose AI obligations effective |
| August 2, 2026 | Full application (all provisions) |

**Risk Categories:**

| Risk Level | Examples | Requirements |
| --- | --- | --- |
| Unacceptable | Social scoring, real-time biometric ID in public, manipulative AI | PROHIBITED |
| High | Employment, education, law enforcement, critical infrastructure | Full compliance requirements |
| Limited | Chatbots, emotion recognition, deepfakes | Transparency obligations |
| Minimal | AI-enabled video games, spam filters | No mandatory requirements |

**High-Risk AI Requirements:**

| Requirement | Details |
| --- | --- |
| Risk Management System | Identify and mitigate risks throughout lifecycle |
| Data Governance | Quality criteria for training, validation, testing data |
| Technical Documentation | Detailed system documentation |
| Record-Keeping | Automatic logging of operation |
| Transparency | Clear information to users |
| Human Oversight | Allow human intervention and override |
| Accuracy and Robustness | Appropriate levels of accuracy, cybersecurity |

**Penalties:**

| Violation | Maximum Penalty |
| --- | --- |
| Prohibited AI | EUR 35 million or 7% of global turnover |
| High-risk non-compliance | EUR 15 million or 3% of global turnover |
| Incorrect information | EUR 7.5 million or 1% of global turnover |

## 4. Data Regulations

### Data Act

**Applicability:** IoT manufacturers, data holders, data recipients, public sector

**Effective Date:** September 12, 2025

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| User Data Access | Users can access data generated by connected products |
| Data Sharing | Data holders must share data upon user request |
| Third-Party Sharing | Users can share data with third parties |
| Cloud Switching | Enable data portability between cloud providers (no fees by 2027) |
| Interoperability | Standards for data spaces and smart contracts |

---

### Digital Services Act (DSA)

**Applicability:** Digital service providers in EU

**Effective Date:** February 17, 2024 (full application)

**Key Requirements:**

| Requirement | Applicability |
| --- | --- |
| Transparency reports | All intermediaries |
| Terms of service clarity | All intermediaries |
| Notice and action mechanisms | Hosting services |
| Trusted flaggers | Online platforms |
| Risk assessment | VLOPs (45M+ users) |
| Independent audits | VLOPs |

## 5. International Security Certifications

### ISO/IEC 27001:2022

**Applicability:** Any organization (voluntary certification)

**ISMS Requirements (Clauses 4-10):**

| Clause | Requirement |
| --- | --- |
| 4 | Context of the organization |
| 5 | Leadership |
| 6 | Planning |
| 7 | Support |
| 8 | Operation |
| 9 | Performance evaluation |
| 10 | Improvement |

**Annex A Controls (93 Controls in 4 Domains):**

#### Organizational Controls (A.5) - 37 Controls

Key controls include: Policies for information security, Threat intelligence (NEW), Information security in project management, Cloud services (NEW), Incident management

#### People Controls (A.6) - 8 Controls

Key controls include: Screening, Terms of employment, Security awareness training, Remote working

#### Physical Controls (A.7) - 14 Controls

Key controls include: Physical security perimeters, Physical security monitoring (NEW), Equipment protection

#### Technological Controls (A.8) - 34 Controls

Key controls include: Access control, Secure authentication, Protection against malware, Logging, Configuration management (NEW), Data masking (NEW), Data leakage prevention (NEW), Secure coding (NEW)

**Transition Deadline:** October 31, 2025 (from ISO 27001:2013)

---

### ISO/IEC 27701:2019 (Privacy Extension)

**Applicability:** Extension to ISO 27001 for Privacy Information Management

**Key Additions:**

| Focus | Requirements |
| --- | --- |
| PII Controllers | Conditions for collection, use, disclosure |
| PII Processors | Processing obligations |
| Privacy by Design | Integrate privacy into ISMS |
| Data Subject Rights | Implement access, correction, deletion |
| Consent Management | Obtain and manage consent |

**Mapping to GDPR:**

- Helps demonstrate GDPR compliance
- Controller and processor requirements aligned
- Privacy impact assessment integration
