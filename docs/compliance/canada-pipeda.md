# Security & Privacy Compliance Standards - Canada

## Overview

This document provides a comprehensive listing of all security and privacy compliance standards and certifications relevant for Canada.

---

## 1. Federal Privacy Laws

### PIPEDA (Personal Information Protection and Electronic Documents Act)

**Applicability:** Private-sector organizations collecting, using, or disclosing personal information in commercial activities across Canada

**10 Fair Information Principles:**

| Principle | Description |
| --- | --- |
| 1. Accountability | Organization responsible for personal information; designate privacy officer |
| 2. Identifying Purposes | Identify purposes for collection at or before time of collection |
| 3. Consent | Knowledge and consent required (express or implied based on sensitivity) |
| 4. Limiting Collection | Collect only what is necessary for identified purposes |
| 5. Limiting Use, Disclosure, Retention | Use/disclose only for identified purposes; retain only as needed |
| 6. Accuracy | Personal information must be accurate, complete, up-to-date |
| 7. Safeguards | Protect with security safeguards appropriate to sensitivity |
| 8. Openness | Make privacy policies readily available |
| 9. Individual Access | Inform individuals of existence, use, disclosure; provide access |
| 10. Challenging Compliance | Ability to challenge compliance to privacy officer |

**Breach Notification Requirements:**

- Report breaches posing "real risk of significant harm" to Privacy Commissioner
- Notify affected individuals
- Maintain breach records for 24 months

**Penalties:** Up to CA$100,000 per violation for certain offenses

**Exemptions:**

- Organizations in provinces with "substantially similar" laws (Quebec, BC, Alberta)
- Employee information in federally-regulated industries
- Personal purposes (non-commercial)

---

### Upcoming: CPPA (Consumer Privacy Protection Act)

**Status:** Proposed replacement for PIPEDA (Bill C-27)

**Key Proposed Changes:**

| Feature | Details |
| --- | --- |
| Right to Data Portability | Transfer data to other organizations |
| Enhanced Deletion Rights | Stronger than current PIPEDA |
| Algorithmic Transparency | Explanation of automated decision-making |
| De-identification Standards | Clear rules for anonymization |
| Private Right of Action | Individuals can sue for violations |
| Penalties | Up to 5% of global revenue or CA$25 million |

**New Elements:**

- Personal Information and Data Protection Tribunal
- Artificial Intelligence and Data Act (AIDA) - companion legislation
- Enhanced enforcement powers for Privacy Commissioner

---

### Anti-Spam Legislation (CASL)

**Applicability:** Any organization sending commercial electronic messages to Canadian recipients

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| Consent | Express or implied consent required before sending |
| Identification | Clear identification of sender |
| Unsubscribe Mechanism | Easy and prompt unsubscribe process |
| Contact Information | Valid physical mailing address |

**Penalties:**

- Up to CA$10 million per violation (corporations)
- Up to CA$1 million per violation (individuals)
- Private right of action

## 2. Provincial Privacy Laws

### Quebec - Law 25 (An Act to Modernize Legislative Provisions Respecting the Protection of Personal Information)

**Applicability:** All organizations collecting personal information of Quebec residents (private and public sector)

**Effective Dates:**

- September 22, 2022: Initial provisions
- September 22, 2023: Majority of provisions
- September 22, 2024: All provisions in force

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| Privacy Officer | Mandatory designation (title and contact must be published) |
| Privacy Impact Assessments | Required for projects involving personal information |
| Breach Notification | 72 hours to notify Commission d'acces a l'information if risk of serious injury |
| Consent | Clear, free, informed consent; separate from other info |
| Transparency | Privacy policy publicly available |
| Cross-Border Transfers | Privacy impact assessment required |
| Right to Data Portability | Provide data in common technological format |
| Right to De-indexation | Request removal from search results |
| Automated Decision-Making | Inform individuals when used, provide recourse |
| Biometric Data | Prior disclosure to CAI before implementing biometric database |
| Default Privacy Settings | Products/services must have highest privacy by default |

**Consent Requirements (Detailed):**

| Type | Requirements |
| --- | --- |
| Express Consent | Required for sensitive information |
| Opt-In | Default for new data collection purposes |
| Separate Consent | Must be obtained separately from other agreements |
| Child Consent | Parental consent for minors under 14 |

**Penalties:**

- Administrative monetary penalties: Up to CA$10 million or 2% of worldwide turnover
- Penal offenses: Up to CA$25 million or 4% of worldwide turnover

---

### British Columbia - PIPA (Personal Information Protection Act)

**Applicability:** Private organizations in BC

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| Consent | Required before collection (opt-out for non-sensitive) |
| Reasonable Purpose | Collection must be reasonable |
| Access Rights | Individuals can access their information |
| Correction Rights | Right to request corrections |
| Privacy Officer | Designate responsible individual |
| Breach Notification | Report to Commissioner if risk of significant harm |
| Retention Limits | Keep only as long as necessary |
| Safeguards | Reasonable security measures |

**Consent Types:**

| Type | When Applicable |
| --- | --- |
| Express Consent | Sensitive information |
| Implied Consent | Non-sensitive, reasonable expectations |
| Deemed Consent | Specific circumstances defined in Act |
| Opt-Out | Limited circumstances |

**Penalties:** Up to CA$100,000 for obstruction of investigation

---

### Alberta - PIPA (Personal Information Protection Act)

**Applicability:** Private organizations in Alberta

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| Consent | Express or implied based on context |
| Notice | Purpose must be identified |
| Access | Right to access personal information |
| Correction | Right to request amendments |
| Breach Notification | Notify Commissioner if real risk of significant harm |
| Safeguards | Reasonable security measures |
| Retention | Destroy when no longer needed |
| Accuracy | Keep information accurate and complete |

**Breach Notification Details:**

- Notify Commissioner without unreasonable delay
- Notify affected individuals if Commissioner directs
- Risk assessment required
- Records must be maintained

**Penalties:** Up to CA$100,000 per offense

## 3. Provincial Health Privacy Laws

### Ontario - PHIPA (Personal Health Information Protection Act)

**Applicability:** Health information custodians in Ontario

**Covered Entities (Health Information Custodians):**

- Healthcare practitioners
- Hospitals
- Long-term care facilities
- Pharmacies
- Laboratories
- Community health centers
- Public health authorities

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| Consent | For collection, use, disclosure of PHI |
| Access | Individuals can access their health records |
| Correction | Right to request corrections |
| Breach Notification | Report if theft, loss, unauthorized access |
| Security | Physical, technical, administrative safeguards |
| Record Retention | Minimum retention periods |
| Agents | Written agreements with agents |
| Electronic Health Records | Specific requirements for EHRs |

**Consent Framework:**

| Type | When Required |
| --- | --- |
| Express Consent | Generally required for collection, use, disclosure |
| Implied Consent | Circle of care (healthcare providers involved in care) |
| No Consent | Legally required disclosures, public health, research (with ethics approval) |

**Penalties:**

- Up to CA$1 million for organizations
- Up to CA$500,000 for individuals
- Additional CA$500,000 AMPs

---

### Alberta - HIA (Health Information Act)

**Applicability:** Health information custodians in Alberta

**Covered Entities:**

- Alberta Health Services
- Hospitals
- Nursing homes
- Pharmacies
- Healthcare practitioners
- Government health programs

**Key Requirements:**

| Requirement | Details |
| --- | --- |
| Collection | Only for authorized purposes |
| Use | Within organization for authorized purposes |
| Disclosure | With consent or as authorized by law |
| Access | Individual right to access health records |
| Amendment | Right to request corrections |
| Security | Administrative, technical, physical safeguards |

**Unique Features:**

- Health Information Regulations provide detailed guidance
- Mandatory breach reporting to Commissioner
- Electronic health record provisions
- Research access framework

**Penalties:** Up to CA$500,000 for offenses

## 4. Canadian Security Standards & Certifications

### CyberSecure Canada

**Applicability:** Voluntary certification for small and medium organizations

**Program Overview:**

- Government of Canada certification program
- Based on Canadian Centre for Cyber Security baseline controls
- Designed for SMBs
- Third-party certification available

**13 Security Controls:**

| Control | Requirements |
| --- | --- |
| 1. Incident Response Plan | Documented plan, contact information, tested procedures |
| 2. Automatic Patching | Automated updates for OS and applications |
| 3. Secure Cloud/Website Configuration | Secure hosting, HTTPS, access controls |
| 4. Data Security | Backup, encryption, disposal procedures |
| 5. Secure Mobile Devices | MDM, encryption, remote wipe capability |
| 6. Backups and Encryption | Regular backups, encryption at rest |
| 7. Strong User Authentication | MFA, password policies, account lockout |
| 8. Access Control | Least privilege, role-based access |
| 9. Perimeter Defense | Firewalls, intrusion detection |
| 10. Portable Media | USB controls, encryption |
| 11. Security Awareness Training | Annual training, phishing awareness |
| 12. Secure Remote Access | VPN, encrypted connections |
| 13. Third-Party Security | Vendor assessment, contracts |

**Certification Process:**

1. Self-assessment against 13 controls
2. Address gaps
3. Third-party assessment (optional)
4. Certification valid for 2 years
